# SmartPark Web UI Mandates

- **Consistent Button Sizing:** Ensure all buttons maintain a consistent size within their context (e.g., buttons in a row should typically have equal widths or a consistent `min-width`).
- **Preserve Original Formatting & Features:** When working on UI components, rigorously adhere to the existing styling, layout patterns, and features established in `App.css`, `index.css`, and existing components.
- **Button Classes:** Prefer using the standard `.primary-btn`, `.secondary-btn`, `.danger-btn`, and `.ghost-btn` classes defined in `App.css`. Avoid inconsistent inline styling for buttons.
